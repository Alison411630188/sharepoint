# SharePoint Power Automate 門戶設計方案

## 設計背景
根據 GitHub 倉庫 `power-automate-new` 的內容，我們需要將 Power Automate 教學網站轉化為企業內部協作風格的 SharePoint 門戶。SharePoint 作為企業文件管理與協作平台，應該呈現專業、可信任、易於導航的視覺語言。

---

## 設計方案一：現代企業風格 (Modern Enterprise)
**設計運動**：Microsoft Fluent Design System 與企業 SaaS 美學的融合

**核心原則**：
1. 清晰的資訊層級 - 使用卡片、分類標籤、進度指示器來組織複雜資訊
2. 微妙的互動反饋 - 按鈕、連結的懸停效果應該溫和而不張揚
3. 信任感與專業性 - 使用灰藍色系與適度的留白營造企業感
4. 易於掃描 - 標題、副標題、圖示的組合讓使用者快速定位

**色彩哲學**：
- 主色：深藍色 (#1F5BA8) - 代表專業、可信任、企業級
- 輔色：淺灰藍 (#E8EEF5) - 背景與卡片區隔
- 強調色：翠綠色 (#10B981) - 成功狀態、行動呼籲
- 文字：深灰 (#1F2937) - 高對比度、易讀性

**版面範例**：
- 頂部導航：固定式，包含 Logo、主選單、搜尋、使用者選單
- 側邊欄：可折疊式，展示內容分類（知識庫、案例庫、文件中心等）
- 主要內容區：寬敞的卡片網格，每個卡片代表一個學習模組或案例
- 頁腳：簡潔的連結與版權資訊

**簽名元素**：
1. 卡片設計 - 帶有細微陰影與邊框的白色卡片
2. 標籤系統 - 彩色標籤標示難度、類別、狀態
3. 進度指示器 - 視覺化學習進度或流程步驟

**互動哲學**：
- 懸停效果：卡片輕微上升、陰影加深
- 點擊反饋：按鈕顏色變深、加載動畫
- 過渡效果：平滑的淡入淡出，無突兀感

**動畫指南**：
- 頁面進入：輕微的上升淡入 (200ms)
- 卡片懸停：陰影增強、Y軸微微上移 (150ms)
- 模態框：從中心縮放進入 (250ms)
- 列表項目：交錯進入動畫 (50ms 間隔)

**排版系統**：
- 標題字體：Segoe UI / 微軟雅黑 (700 weight) - 企業級標準字體
- 正文字體：Segoe UI / 微軟雅黑 (400 weight)
- 標題大小：H1 2.5rem, H2 1.875rem, H3 1.25rem
- 行高：1.6 (正文)、1.4 (標題)

**概率**：0.08

## 設計方案二：極簡知識管理風格 (Minimalist Knowledge Hub)
**設計運動**：北歐極簡設計 + 知識管理系統的簡潔美學

**核心原則**：
1. 內容優先 - 最小化視覺干擾，讓內容成為焦點
2. 單色系為主 - 灰階為基礎，單一強調色
3. 充足的呼吸空間 - 大量留白，讓眼睛舒適
4. 一致的網格系統 - 所有元素對齊到統一的模組化網格

**色彩哲學**：
- 主色：中性灰 (#4B5563) - 沉靜、專業、不分散注意力
- 背景：淺灰白 (#F9FAFB) - 柔和、減少眼睛疲勞
- 強調色：單一亮色 (#FF6B35 或 #0066CC) - 行動呼籲
- 邊界：淺灰 (#E5E7EB) - 細微的視覺分隔

**版面範例**：
- 頂部導航：極簡設計，只顯示必要項目
- 主要內容：垂直排列的列表或單欄網格
- 側邊欄：隱藏式，需要時才展開
- 卡片設計：無邊框、無陰影，只用背景色區隔

**簽名元素**：
1. 細線邊框 - 1px 灰色邊框定義區域
2. 序號系統 - 數字或圖示標示步驟
3. 最小化圖示 - 簡潔的線性圖示，不填充

**互動哲學**：
- 懸停效果：背景色微微變深，無動畫
- 點擊反饋：簡單的顏色變化
- 過渡效果：快速、不明顯的淡入淡出

**動畫指南**：
- 頁面進入：無動畫或極簡淡入 (100ms)
- 卡片懸停：背景色變化 (100ms)
- 模態框：直接出現或簡單淡入
- 列表項目：無交錯，同時出現

**排版系統**：
- 標題字體：系統字體 (Segoe UI / -apple-system) 400 weight
- 正文字體：系統字體 400 weight
- 標題大小：H1 2rem, H2 1.5rem, H3 1rem
- 行高：1.5 (正文)、1.3 (標題)

**概率**：0.07

## 設計方案三：活力協作風格 (Vibrant Collaboration)
**設計運動**：現代 SaaS 產品設計 + 協作工具的友善美學

**核心原則**：
1. 色彩豐富 - 多色系搭配，營造活力與創意氛圍
2. 圓潤柔和 - 大圓角、柔和邊界，降低視覺銳利感
3. 視覺故事 - 插圖、圖示、漸層講述故事
4. 互動樂趣 - 豐富的微互動，讓使用者感到愉悅

**色彩哲學**：
- 主色：紫色 (#7C3AED) - 創意、創新、現代
- 輔色組：
  - 青色 (#06B6D4) - 信息、協作
  - 粉紅色 (#EC4899) - 活力、熱情
  - 琥珀色 (#F59E0B) - 警告、重點
- 背景：淡紫白 (#F3E8FF) - 溫暖、邀請性

**版面範例**：
- 頂部導航：彩色漸層背景，圓潤按鈕
- 側邊欄：彩色標籤分類，每個分類不同顏色
- 主要內容：彩色卡片網格，每個卡片有不同的強調色
- 頁腳：彩色漸層背景

**簽名元素**：
1. 彩色標籤 - 每個分類或狀態有獨特顏色
2. 圓形圖示 - 圓潤的圖示容器
3. 漸層背景 - 微妙的漸層增加深度

**互動哲學**：
- 懸停效果：卡片旋轉微角、陰影加深、顏色飽和度增加
- 點擊反饋：按鈕有彈跳感
- 過渡效果：平滑的彈性動畫

**動畫指南**：
- 頁面進入：卡片從下方彈入 (300ms cubic-bezier)
- 卡片懸停：微妙旋轉 (2-3 度) + 上升 (150ms)
- 模態框：從中心放大進入，帶有彈性 (250ms)
- 列表項目：交錯彈入 (80ms 間隔)

**排版系統**：
- 標題字體：Poppins / 微軟雅黑 (600 weight) - 現代、友善
- 正文字體：Inter / 微軟雅黑 (400 weight)
- 標題大小：H1 2.75rem, H2 2rem, H3 1.375rem
- 行高：1.7 (正文)、1.3 (標題)

**概率**：0.06

## 最終選擇
**選定方案：現代企業風格 (Modern Enterprise)**

此方案最適合 SharePoint 門戶的場景，因為：
1. **企業級信任感** - 深藍色與灰藍色組合傳達專業性
2. **清晰的資訊架構** - 卡片與標籤系統便於組織複雜的教學內容
3. **微妙的互動** - 不過度設計，保持焦點在內容上
4. **易於擴展** - 標籤系統與卡片設計便於未來新增內容

### 實施細節
- **主色調**：深藍色 (#1F5BA8) 用於導航、按鈕、強調
- **背景**：淺灰藍 (#F8FAFC) 用於次要區域
- **文字**：深灰 (#1F2937) 用於正文
- **成功狀態**：翠綠色 (#10B981)
- **字體**：Segoe UI / 微軟雅黑 (企業標準)
- **圓角**：中等 (0.5rem - 0.75rem)
- **陰影**：細微 (0 1px 3px rgba)
